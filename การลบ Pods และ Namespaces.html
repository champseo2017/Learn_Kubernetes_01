การลบ Pods และ Namespaces

สรุปการจัดการทรัพยากรด้วย Label และ Namespace

สรุปจากรูปภาพ: การใช้ Label Selector

ในระบบที่มี Pods จำนวนมาก การจัดกลุ่มด้วย Labels เป็นสิ่งสำคัญมาก โดยมักจะแบ่งตาม 2 ประเภทหลัก:

  • app: ใช้บอกว่าเป็น Pod ของ service ไหน (เช่น Account Service, Product Catalog)
  • rel: ใช้บอกว่าเป็น release เวอร์ชั่นไหน (เช่น stable, beta, หรือ canary)

เราสามารถใช้ Label Selector เช่น rel=canary เพื่อเลือกและลบ Pods ทั้งหมดที่เป็นเวอร์ชันทดสอบได้พร้อมกันในครั้งเดียว โดยไม่กระทบกับ Pods ที่เป็นเวอร์ชันอื่นเลย

การลบ Pods โดยการลบ Namespace

สำหรับ Pod ที่อยู่ใน custom-namespace หากเราไม่ต้องการใช้งานทั้ง Pods และ Namespace นั้นอีกต่อไป วิธีที่ง่ายและรวดเร็วที่สุดคือการลบ Namespace ทั้งหมดทิ้งไป

เมื่อเราสั่งลบ Namespace ทรัพยากรทุกอย่างที่อยู่ข้างใน (เช่น Pods, Services) จะถูกลบตามไปด้วยโดยอัตโนมัติ ซึ่งเป็นวิธีที่สะดวกมากในการเคลียร์สภาพแวดล้อมที่ไม่ใช้แล้ว

ตัวอย่างคำสั่ง

# สั่งลบ namespace ที่ชื่อว่า "custom-namespace"
# ns เป็นคำย่อของ namespaces
# เมื่อรันคำสั่งนี้ ทุกอย่างที่อยู่ใน namespace นี้จะถูกลบไปด้วย
$ kubectl delete ns custom-namespace
