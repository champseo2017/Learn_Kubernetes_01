Namespace Isolation ใน Kubernetes

ทำความเข้าใจข้อจำกัดและความสามารถที่แท้จริง

สรุปความสามารถในการแยกส่วน

สิ่งสำคัญที่ต้องเข้าใจคือ namespace ไม่ได้แยกทุกอย่างออกจากกันโดยสมบูรณ์เสมอไป โดยปกติแล้ว namespace จะช่วยแยก objects (เช่น Pod, Service) ออกเป็นกลุ่มๆ ทำให้เราจัดการทรัพยากรได้ง่ายขึ้น แต่ไม่ได้หมายความว่า Pod ที่ทำงานอยู่คนละ namespace จะถูกตัดขาดจากกัน

ความเข้าใจผิดที่พบบ่อย: หลายคนอาจคิดว่าถ้า Pod อยู่คนละ namespace มันจะไม่สามารถสื่อสารกันได้ ซึ่งไม่เป็นความจริงเสมอไปครับ

หัวใจสำคัญคือ: การที่ namespace จะมีการแยกเครือข่าย (network isolation) หรือไม่นั้น ขึ้นอยู่กับ "โซลูชันเน็ตเวิร์ก" (networking solution) ที่ใช้กับ Kubernetes Cluster ของเราครับ ถ้าโซลูชันนั้นไม่ได้ตั้งค่าให้แยกเครือข่ายไว้ Pod จาก namespace "foo" ก็ยังสามารถส่ง traffic ไปหา Pod ใน namespace "bar" ได้สบายๆ แค่รู้ IP address ของปลายทาง
