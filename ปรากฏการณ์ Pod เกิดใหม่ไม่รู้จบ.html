ปรากฏการณ์ Pod เกิดใหม่ไม่รู้จบ

ทำความเข้าใจเบื้องหลังการทำงานของ ReplicationController

สรุปสถานการณ์: ยิ่งลบ ยิ่งเกิด

บางครั้งเมื่อเราต้องการล้าง Pod ทั้งหมดใน Namespace ด้วยคำสั่ง kubectl delete po --all อาจจะพบกับเรื่องน่าประหลาดใจ คือ Pod เก่ากำลังถูกลบ (สถานะ Terminating) แต่กลับมี Pod ตัวใหม่โผล่ขึ้นมาแทนที่ทันที

สาเหตุคืออะไร? ปรากฏการณ์นี้เกิดขึ้นเพราะ Pod นั้นไม่ได้ถูกสร้างขึ้นมาโดยตรง แต่ถูกควบคุมโดย Resource อื่นที่ทำงานอยู่เบื้องหลัง เช่น ReplicationController

หน้าที่ของ ReplicationController คือการ "ดูแล" ให้มีจำนวน Pod ที่ต้องการทำงานอยู่ตลอดเวลา เมื่อมันตรวจพบว่า Pod ที่ดูแลอยู่หายไป (เพราะเราลบ) มันก็จะสร้างตัวใหม่ขึ้นมาทดแทนทันที ดังนั้น หากต้องการลบ Pod ให้หายถาวร เราต้องไปลบที่ตัว Controller ครับ

ภาพประกอบการทำงาน

ลำดับคำสั่งและผลลัพธ์

# 1. ตรวจสอบ Pod ที่กำลังทำงานอยู่
$ kubectl get pods
NAME          READY   STATUS    RESTARTS   AGE
kubia-zxzij   1/1     Running   0          1d

# 2. สั่งลบ Pod ทั้งหมด
$ kubectl delete po --all
pod "kubia-zxzij" deleted

# 3. ตรวจสอบอีกครั้งและพบว่ามี Pod ใหม่เกิดขึ้น!
$ kubectl get pods
NAME          READY   STATUS        RESTARTS   AGE
kubia-09as0   1/1     Running       0          1d
kubia-zxzij   1/1     Terminating   0          1d
