Replication และ Controllers

สรุปการ Deploy Pods แบบอัตโนมัติ

เปลี่ยนสู่ระบบอัตโนมัติ

ในบทนี้เราจะเปลี่ยนจากการสร้างและจัดการ pods ด้วยตนเอง (manual) ไปสู่การทำให้ทุกอย่างเป็นอัตโนมัติ เพื่อให้ deployments ของเราทำงานได้อย่างต่อเนื่องและมีสถานะ healthy โดยไม่ต้องเข้าไปดูแลเอง หัวใจสำคัญคือ เราแทบจะไม่สร้าง pods ขึ้นมาตรงๆ แต่จะสร้าง resource ที่สูงกว่าเช่น ReplicationControllers หรือ Deployments ให้ทำหน้าที่สร้างและจัดการ pods แทนเรา

Unmanaged vs Managed Pods

Pods ธรรมดา (Unmanaged)

ถ้า container ล่ม Kubernetes จะรีสตาร์ทให้ แต่ถ้าหาก node ที่ Pod รันอยู่ล่ม Pod นั้นจะหายไปเลยและไม่ถูกสร้างขึ้นมาใหม่

Pods ที่ถูกจัดการ (Managed)

Controller จะคอยดูแลอยู่เสมอ ถ้า node ล่ม มันจะสร้าง Pod ใหม่ขึ้นมาทดแทนบน node อื่นที่ยังทำงานได้โดยอัตโนมัติ

เนื้อหาที่จะได้เรียนรู้

  • การดูแล pods ให้มีสถานะ healthy
  • การรัน pod แบบเดียวกันหลาย instance
  • การย้าย pod อัตโนมัติเมื่อ node ล่ม
  • การทำ Horizontal Scaling
  • การรัน system-level pods
  • การรัน batch jobs
  • การตั้งเวลาให้ job ทำงาน
