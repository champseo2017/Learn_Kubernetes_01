สรุป Kubernetes: การตรวจสอบ Pod บน Node ไหน

แนวคิดหลัก

ใน Kubernetes, Pod จะถูก schedule ไปยัง Node ที่มี CPU/Memory เพียงพอ ไม่สำคัญว่า Node ไหน เพราะ:

  • ทุก Pod มี Environment OS เดียวกัน
  • Pod มี IP เอง สามารถสื่อสารข้าม Node ได้
  • Resources ถูก allocate ตาม request ไม่ขึ้นกับ Node

คลิกปุ่มด้านล่างเพื่อ simulate การ schedule Pod

แสดง Pod IP และ Node

ปกติ kubectl get pods ไม่แสดง Node/IP แต่ใช้ -o wide เพื่อดูเพิ่ม: 

$ kubectl get pods -o wide
NAME        READY   STATUS    RESTARTS   AGE   IP         NODE
kubia-hczji 1/1     Running   0          7s    10.1.0.2   gke-kubia-85...
